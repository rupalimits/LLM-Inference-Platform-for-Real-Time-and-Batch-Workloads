# Cloud-Native LLM Inference Platform

A production-ready, cloud-native platform for real-time and batch LLM inference built on **vLLM**, **FastAPI**, **Celery**, **Kubernetes**, and a full **Prometheus + Grafana** observability stack.

---

## Architecture

```
┌─────────────────────────────────────────────────────┐
│                    Clients                          │
└────────────┬────────────────────────┬───────────────┘
             │ REST / SSE             │ Batch API
             ▼                        ▼
┌────────────────────┐     ┌──────────────────────┐
│  FastAPI Server    │     │   FastAPI /v1/batch  │
│  (vLLM AsyncEngine)│     │   → Celery task      │
│  /v1/inference     │     │   → Redis queue      │
│  /v1/inference/    │     └──────────┬───────────┘
│    stream          │                │
└────────┬───────────┘     ┌──────────▼───────────┐
         │                 │   Celery Worker       │
         │ Prometheus       │   (vLLM sync engine) │
         ▼                 └──────────────────────┘
┌────────────────────┐
│   Prometheus       │◄── scrapes /metrics on
│   + Grafana        │    inference:8000 & worker:9091
└────────────────────┘
```

## Quick Start (Docker Compose)

```bash
# 1. Configure environment
cp .env.example .env
# Edit .env: set MODEL_NAME, HUGGING_FACE_HUB_TOKEN (for gated models), etc.

# 2. Build images
./scripts/build.sh

# 3. Launch the full stack
docker compose up -d

# 4. Check health
curl http://localhost:8000/health
```

Services available:
- **Inference API**: http://localhost:8000/docs
- **Prometheus**: http://localhost:9092
- **Grafana**: http://localhost:3000 (admin / admin)

## API Reference

### Real-time Inference

```bash
curl -X POST http://localhost:8000/v1/inference \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Explain gradient descent in simple terms.",
    "max_tokens": 256,
    "temperature": 0.7
  }'
```

### Streaming Inference (SSE)

```bash
curl -X POST http://localhost:8000/v1/inference/stream \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Tell me a story.", "max_tokens": 512}'
```

### Batch Inference

```bash
# Submit
JOB=$(curl -s -X POST http://localhost:8000/v1/batch \
  -H "Content-Type: application/json" \
  -d '{"prompts": ["Prompt A", "Prompt B"], "max_tokens": 128}' | jq -r .job_id)

# Poll status
curl http://localhost:8000/v1/batch/$JOB
```

## Kubernetes Deployment

### Raw manifests

```bash
# Apply all manifests
./scripts/deploy.sh

# With Helm
USE_HELM=true ./scripts/deploy.sh
```

### Helm only

```bash
helm upgrade --install llm-platform ./helm \
  --namespace llm-platform \
  --create-namespace \
  --set model.name=meta-llama/Llama-2-7b-chat-hf \
  --set hfTokenSecret=hf-secret
```

### HuggingFace token secret (gated models)

```bash
kubectl create secret generic hf-secret \
  --from-literal=token=<YOUR_HF_TOKEN> \
  -n llm-platform
```

## Load Testing

```bash
# Install deps and run 2-minute test with 20 users
./scripts/load_test.sh

# Or with custom settings
HOST=http://localhost:8000 USERS=50 SPAWN_RATE=5 RUN_TIME=5m ./scripts/load_test.sh
```

The script produces:
- `load-tests/results/locust_stats.csv` — per-endpoint stats
- `load-tests/results/locust_stats_history.csv` — time-series RPS
- `load-tests/results/report.md` — human-readable p50/p95/p99 report

## Observability

### Prometheus metrics exposed

| Metric | Type | Description |
|--------|------|-------------|
| `inference_requests_total` | Counter | Requests by endpoint and status code |
| `inference_request_duration_seconds` | Histogram | Request latency (p50/p95/p99 via PromQL) |
| `inference_active_requests` | Gauge | Concurrent in-flight requests |
| `inference_tokens_generated_total` | Counter | Tokens generated per model |
| `batch_jobs_total` | Counter | Batch jobs by status |
| `worker_batch_task_duration_seconds` | Histogram | Worker task duration |
| `worker_tokens_generated_total` | Counter | Tokens generated by batch worker |

### Grafana dashboard

Import `monitoring/grafana/dashboards/inference.json` or let auto-provisioning load it.

Panels included:
- Request rate per endpoint
- p50 / p95 / p99 latency
- Active requests gauge
- Tokens/second
- 5xx error rate
- Batch job rate
- Worker task duration

## Switching Models

Change the `MODEL_NAME` environment variable to any HuggingFace model ID supported by vLLM:

```bash
# docker-compose
MODEL_NAME=mistralai/Mistral-7B-Instruct-v0.2 docker compose up -d

# Helm
helm upgrade llm-platform ./helm --set model.name=mistralai/Mistral-7B-Instruct-v0.2
```
